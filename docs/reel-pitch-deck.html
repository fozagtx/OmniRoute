The product

Reel is a result bounty board for clippers.

02PostClipper submits a public YouTube link.
03CheckAgent reads the visible result.
04PayEscrow releases reward when the target is hit.
How it works

One clean loop: funded work, public proof, automatic payout.

For brands

Only pay for clips that reach the bar.

Clear spendSet the view target, payout, max winners, and deadline before launch.
Visible progressSee submitted links, checks, paid clips, and unused funds.
Refund unused fundsClose a bounty and recover funds that were not paid out.
YouTube firstStart where clip work is already obvious, then expand to more public platforms.
For clippers

Pick paid work, post the link, get paid when it hits.

Funded before workClippers can see a bounty is backed before spending time on it.
Public link proofSubmit a YouTube URL instead of arguing over screenshots.
Direct payoutWhen the visible views pass the target, reward goes to the clipper wallet.
Why the agent matters

The agent is the checker, not the boss.

  • It reads the public clip page.
  • It returns the visible view count.
  • It can check again later if the clip is still growing.
  • It does not need a creator login.
What is live

Reel is already running as a working testnet product.

Brand and clipper workspaces are separate. YouTube checks are live first. Scheduled rechecks are wired for clips that need more time.
